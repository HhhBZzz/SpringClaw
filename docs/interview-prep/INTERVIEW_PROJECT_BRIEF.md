# INTERVIEW_PROJECT_BRIEF

## 一句话介绍

这是一个基于 Spring Boot 3 + Spring AI 的 AI Agent 项目，核心是 OPAR 闭环（观察-计划-行动-反思），支持聊天、工具调用和记忆落库。

## 你在项目中做了什么（面试可直接说）

1. 我把 Agent 主流程拆成 OPAR 四阶段，避免“一次性问答”不可控。
2. 我用 Spring AI 的 `@Tool` 做工具调用，并用 AOP 统一做限流和审计。
3. 我把记忆拆成双轨：MySQL 存事件流（可审计），Redis VectorStore 做语义召回（可扩展）。

## 项目最核心的 3 个功能

1. 聊天接口：`/api/chat/send`、`/api/chat/stream`
2. 工具能力：模型可调用系统时间/文件读写等工具
3. 数据留痕：会话和事件写入 MySQL，可做问题追踪和复盘

## 面试亮点（背诵版）

1. 我做了一个“可落地”的 Agent，不只是调用大模型接口，而是有完整执行闭环和降级策略。
2. 我把工具调用做成统一治理，解决了模型乱调用工具带来的稳定性风险。
3. 我做了可审计的记忆系统，能回答“模型为什么这么回答”，这在企业场景很关键。

## 面试官常问问题（短答）

1. 为什么不用一次性问答？
   - 一次性问答不可控，OPAR 能把规划、执行、反思拆开，便于审计和降级。
2. 工具怎么防止乱用？
   - 通过 AOP 拦截 `@Tool`，统一限流、记录调用结果，异常可追踪。
3. 记忆为什么要 MySQL + Vector？
   - MySQL 负责“可追责”，Vector 负责“语义检索”，两者互补。
